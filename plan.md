# 角色设定
你是一位经验丰富的全栈工程师，擅长使用 Python + LangChain 开发智能客服系统。请帮我开发一个极简版智能工单系统。

# 项目需求
开发一个基于 LangChain 的智能客服工单系统，支持三个核心功能：
1. 订单物流查询 - 用户输入订单号，查询实时物流状态
2. 催单 - 用户催促订单处理，系统创建工单通知客服
3. 退单 - 用户取消订单，系统校验并处理退款

# 技术栈要求
- 后端：Python 3.9+
- AI框架：LangChain + OpenAI API (或 其他LLM)
- 数据存储：内存存储即可（用Dict模拟），预留数据库接口
- Web界面：Gradio（简单快速）或 FastAPI + 前端
- 意图识别：必须用大模型做意图分类，不要硬编码规则

# 核心业务流程

## 1. 意图识别模块
- 使用 LangChain + Pydantic 做结构化输出
- 识别三种意图：logistics(物流查询)/urgent(催单)/cancel(退单)
- 提取实体：订单号(格式ORD+数字)、用户诉求细节
- 置信度低于0.7时，返回澄清问题

## 2. 物流查询模块
- 输入：订单号
- 处理：查询订单状态 → 如已发货则查询物流轨迹
- 输出：格式化物流信息（最新状态、预计送达、最近3条轨迹）

## 3. 催单模块
- 输入：多轮对话收集 订单号 + 催单原因（可选）
- 处理：创建工单(Ticket) → 保存到存储 → 通知客服（打印日志模拟）
- 输出：工单号 + 预计处理时间 + 客服联系方式

## 4. 退单模块
- 输入：多轮对话收集 订单号 + 退单原因
- 校验规则：
  - 已取消：拒绝，提示已取消
  - 已签收：拒绝，提示走售后退货流程
  - 其他状态：允许取消
- 处理：调用取消API → 更新订单状态 → 触发退款
- 输出：取消结果 + 退款金额 + 到账时间
